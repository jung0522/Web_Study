패킷은 네트워크를 통해 데이터를 전달하기 위해 나눈 작은 단위의 데이터 블록

TCP(Transmission Control Protocol): 네트워크 통신 규칙

IP (Internet Protocol):
인터넷 계층에서 데이터를 목적지까지 전송하는 역할을 하며, 
경로를 선택(라우팅)하고 패킷을 전송한다.

	- IPv4: 32비트, 4개의 8비트(즉, 0부터 255까지의 숫자 4개로 구성된 주소)
          예: 192.168.0.1, 10.0.0.5

        - IPv6: 128비트, 8개의 16비트 숫자(16진수로 표시)
          예: 2001:0db8:85a3:0000:0000:8a2e:0370:7334



활용 사례

TCP: 웹 브라우저(HTTP, HTTPS), 이메일(SMTP), 파일 전송(FTP).
IP: 모든 네트워크 기반 통신의 기반으로 동작.

예시

IP: 데이터를 목적지로 보내는 택배회사 (주소 확인 및 운송 담당).
TCP: 데이터의 손상, 손실, 순서 문제를 확인하고 해결하는 배송 품질 관리.

TCP/IP는 주로 4계층 모델

1. 애플리케이션 계층 (Application Layer)
사용자와 직접 상호작용하는 응용 프로그램의 데이터 처리를 담당.

주요 프로토콜:
	HTTP/HTTPS: 웹 데이터 전송
	SMTP: 이메일 전송

2. 전송 계층 (Transport Layer)
데이터를 송신자와 수신자 간에 신뢰성 있게 전송하는 역할을 한다.
주요 프로토콜:
	TCP (Transmission Control Protocol):
		연결 지향적
		데이터의 신뢰성, 순서 보장
	UDP (User Datagram Protocol):
		비연결적
		빠른 전송, 신뢰성은 보장하지 않음

	TCP 활용 사례
		웹 브라우징 (HTTP/HTTPS)
		이메일 (SMTP, IMAP, POP3)
		파일 전송 (FTP)
		원격 접속 (SSH, Telnet)

	UDP 활용 사례
		실시간 스트리밍 (동영상, 오디오)
		온라인 게임

3. 인터넷 계층 (Internet Layer)
데이터를 패킷 단위로 나누고, 최적의 경로를 통해 목적지에 전달.
	주요 프로토콜:
		IP (Internet Protocol):
		데이터의 전송 경로 지정 (라우팅)

4. 네트워크 액세스 계층 (Network Access Layer)
실제로 데이터를 물리적 네트워크를 통해 전송.
	주요 프로토콜:
		Ethernet, Wi-Fi 등


AWS(Amazon Web Service): Amazon이 제공하는 클라우드 컴퓨팅 플랫폼

VPC는 가상 사설 네트워크를 구성할 수 있게 해주는 AWS(Amazon Web Services)
- 원칙적으로 외부 접근 제한하는 가상의 데이터 센터
- 외부에 격리된 네트워크 컨테이너(도커) 구성 가능
- 리전 단위

	활용 사례
        -EC2, RDS

	서브넷 (Subnet)은 VPC 내에서 네트워크(VPC에 할당된 IP)를 세분화한 단위
        - 하나의 서브넷은 하나의 가용영역(AZ) 안에 위치

	- 서브넷의 IP 주소 범위

		서브넷은 CIDR (Classless Inter-Domain Routing) 사이다 표기법을 사용하여 		IP 주소 범위를 정의한다. 예를 들어, 10.0.0.0/24는 10.0.0.0부터 10.0.0.255까		지의 IP 주소를 포함하는 서브넷이다.

	퍼블릭 서브넷과 프라이빗 서브넷

	퍼블릭 서브넷 (Public Subnet)

		퍼블릭 서브넷에 배치된 리소스는 인터넷과 직접 통신할 수 있다. 일반적으		로 인터넷 게이트웨이를 통해 인터넷과 연결된다.

		사용 예시: 웹 서버, 로드 밸런서 등 외부와 직접 상호작용이 필요한 리소스			     를 배치.
	
	프라이빗 서브넷 (Private Subnet)

		프라이빗 서브넷에 배치된 리소스는 인터넷과의 직접적인 연결이 없다. 이 		서브넷에 배치된 리소스는 NAT 게이트웨이나 NAT 인스턴스를 사용하여 인		터넷에 간접적으로 접근할 수 있다.

		사용 예시: 데이터베이스 서버, 애플리케이션 서버 등 외부와의 직접적인 연			     결이 필요 없는 리소스를 배치.


	서브넷의 IP 개수
	x.x.x.x/n (n: 서브넷 마스크)
	서브넷 크기 = 2^(32 - n) (총 IP 개수)
	네트워크 주소와 브로드캐스트 주소가 예약되어 있으므로 실제로 사용 가능한 IP 	주소는 총 IP 개수에서 2개를 뺀 값

	AWS 서브넷의 IP 개수
        ex)10.0.0.0/24
        10.0.0.0: 네트워크 주소
        10.0.0.1: VPC Router
        10.0.0.2: DNS Server
        10.0.0.3: 미래 사용을 위해 남겨 둠
        10.0.0.255(마지막 번호): 브로드캐스트 주소
        - 2^(32-24) - 5 = 251
